# 大模型接入规划

## 一、接入目标

后续项目计划接入大语言模型，使北极熊桌宠不仅能展示和互动，还能进行智能对话、学习陪伴、任务提醒和情绪化反馈。大模型不是中期阶段必须完成的功能，但应作为后续创新扩展方向写入设计方案。

## 二、主要应用场景

| 场景 | 说明 |
| --- | --- |
| 智能聊天 | 用户可以和北极熊桌宠进行自然语言对话 |
| 学习陪伴 | 桌宠可以鼓励用户学习、提醒休息、辅助制定计划 |
| 情绪反馈 | 根据用户输入生成不同语气的回复 |
| 任务提醒 | 根据任务和番茄钟状态生成提醒文案 |
| 喂养反馈 | 投喂后生成更自然的感谢或状态反馈 |
| 日志总结 | 根据一天的互动记录生成简短日记或总结 |

## 三、接入方式设计

后续可以采用两种方式：

### 1. 云端大模型接口

通过 API 调用云端大模型，优点是效果更好、回复质量更稳定。适合实现智能聊天、学习陪伴和复杂文本生成。

基本流程：

```text
用户输入 -> PySide6 聊天窗口 -> LLM 服务适配层 -> 云端模型 API -> 返回回复 -> 桌宠气泡/聊天窗口显示
```

### 2. 本地大模型

如果考虑离线使用，可以接入本地模型或本地推理服务。优点是隐私性更好，但对电脑性能要求更高。

基本流程：

```text
用户输入 -> PySide6 聊天窗口 -> 本地模型服务 -> 返回回复 -> 桌宠反馈
```

## 四、系统结构扩展

后续可在项目中增加：

```text
src/

│  └─ memory.py            # 简单对话记忆
├─ modules/
│  └─ chat.py              # 聊天/智能陪伴模块
```

## 五、和五大模块的关系

| 模块 | 大模型作用 |
| --- | --- |
| 桌宠形象与桌面交互模块 | 根据用户点击或动作生成自然反馈 |
| 宠物状态与成长任务模块 | 根据状态和任务生成鼓励、提醒和总结 |
| 背包、商店与喂养模块 | 投喂后生成拟人化回复 |
| 系统设置与存档管理模块 | 保存大模型开关、API Key、本地模型路径等配置 |
| 通知日志与智能陪伴模块 | 提供聊天、日记总结、情绪陪伴和任务提醒 |
