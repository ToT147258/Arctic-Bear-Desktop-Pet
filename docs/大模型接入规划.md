# 大模型接入规划与当前实现

## 一、接入目标

项目已接入大语言模型基础能力，使北极熊桌宠不仅能展示和互动，还能进行智能对话、学习陪伴、任务提醒和情绪化反馈。当前实现采用“云端 OpenAI 兼容接口 + 本地兜底回复”的方案，用户未配置 API Key 时仍可正常使用本地轻量聊天。

## 二、主要应用场景

| 场景 | 说明 |
| --- | --- |
| 智能聊天 | 用户可以和北极熊桌宠进行自然语言对话 |
| 学习陪伴 | 桌宠可以鼓励用户学习、提醒休息、辅助制定计划 |
| 情绪反馈 | 根据用户输入生成不同语气的回复 |
| 任务提醒 | 根据任务和番茄钟状态生成提醒文案 |
| 喂养反馈 | 投喂后生成更自然的感谢或状态反馈 |
| 日志总结 | 根据一天的互动记录生成简短日记或总结 |

## 三、接入方式设计

当前采用两种兼容方式：

### 1. 云端大模型接口

通过 API 调用云端大模型，优点是效果更好、回复质量更稳定。当前支持 DeepSeek、ChatGPT/OpenAI、智谱 GLM、通义千问、Kimi 和自定义 OpenAI 兼容接口。

基本流程：

```text
用户输入 -> PySide6 聊天窗口 -> LLM 服务适配层 -> 云端模型 API -> 返回回复 -> 桌宠气泡/聊天窗口显示
```

### 2. 本地兜底回复

如果未启用联网大模型、API Key 未配置或接口调用失败，聊天模块会自动使用本地规则回复，保证桌宠不会因为模型不可用而失去基础陪伴功能。

基本流程：

```text
用户输入 -> PySide6 聊天窗口 -> 本地规则回复 -> 桌宠反馈
```

## 四、系统结构扩展

当前已增加：

```text
src/
├─ llm_client.py           # 大模型统一调用接口、供应商配置、角色提示词
├─ modules/
│  ├─ chat.py              # 聊天/智能陪伴模块
│  └─ settings.py          # AI 大模型配置入口
```

## 五、和五大模块的关系

| 模块 | 大模型作用 |
| --- | --- |
| 桌宠形象与桌面交互模块 | 根据用户点击或动作生成自然反馈 |
| 宠物状态与成长任务模块 | 根据状态和任务生成鼓励、提醒和总结 |
| 背包、商店与喂养模块 | 投喂后生成拟人化回复 |
| 系统设置与存档管理模块 | 保存大模型开关、API Key、本地模型路径等配置 |
| 通知日志与智能陪伴模块 | 提供聊天、日记总结、情绪陪伴和任务提醒 |
