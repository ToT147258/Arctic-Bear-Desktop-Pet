# PolarBear 旧项目格式角色包

这个目录按旧项目 `res/role/角色名/` 的方式组织：

```text
PolarBear/
├─ pet_conf.json
├─ act_conf.json
└─ action/
   ├─ idle_000.png
   ├─ idle_001.png
   ├─ walk_right_000.png
   ├─ walk_right_001.png
   ├─ wave_000.png
   └─ ...
```

程序启动时会优先读取这个角色包。只要动作帧放进 `action/`，就会按旧项目的播放逻辑运行。

## 动作命名

需要的文件名前缀：

- `idle_000.png ...`：待机
- `walk_right_000.png ...`：向右走
- `walk_left_000.png ...`：向左走，可选；没有时也可以只做右走
- `wave_000.png ...`：挥手
- `jump_000.png ...`：跳跃
- `sleep_000.png ...`：睡觉
- `drag_000.png ...`：拖拽

## 推荐帧数

- 待机：12 到 24 帧
- 走路：16 到 32 帧
- 挥手：12 到 24 帧
- 跳跃：12 到 18 帧
- 睡觉：8 到 16 帧
- 拖拽：4 到 8 帧

## 重要说明

旧项目的“生成动画人物方法”本质上不是自动生成，而是定义一个角色包格式，然后把连续 PNG 动作帧放进 `action/`。如果要真实、清晰、无漏洞，动作帧最好来自：

- Blender/3D 模型渲染；
- Live2D/Spine 导出；
- 真实视频抠像后抽帧；
- 专门的视频生成工具输出短片后再拆帧。

普通图片生成很难保证每帧一致，所以不适合作为最终高质量桌宠动作源。
